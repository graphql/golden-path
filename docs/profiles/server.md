Use this profile if you build or operate a GraphQL server/gateway endpoint.

Most organizations should start with the
[First-party API profile (default)](/profiles/server/first-party). It is the right
baseline for the majority of GraphQL deployments where clients are controlled
by the same organization.

Use the [Third-party API profile (opt-in)](/profiles/server/third-party) only when you
must accept arbitrary operations from untrusted external clients.

## Server sub-profiles

- [First-party API (default)](/profiles/server/first-party)
- [Third-party API (opt-in)](/profiles/server/third-party)
